Server

server. Server

Server side entry point for a soundworks application.

This object hosts configuration informations, as well as methods to initialize and start the application. It is also responsible for creating the static file (http) server as well as the socket server.

Constructor

new Server(config)

Source:
  • server/Server.js, line 93
Example
import { Server } from '@soundworks/core/server';
const server = new Server({
  app: {
    name: 'my-example-app',
    clients: { myClient: { target: 'node' } },
  },
  env: {
    port: 8888,
  },
});
await server.init();
await server.start();

Methods

(async) init()

Source:
  • server/Server.js, line 235

Method to be called before start in the initialization lifecycle of the soundworks server.

What it does:

  • prepapre http(s) server and routing according to the informations declared in config.app.clients
  • starts all registered plugins

After await server.init(), you can safely use the StateManager, as well as any registered Plugins.

Example
// defaults to
const server = new Server(config);
await server.init();
await server.start();

If you end up using this, please contact me to explain your use-case :)

setDefaultTemplateConfig()

Source:
  • server/Server.js, line 742

Configure the server to work out-of-the box with the soundworks-template directory tree structure.

  • uses template-literal package as html templateEngine
  • define .build/server/tmpl as the directory in which html template can be found
  • define the clientConfigFunction function that return client compliant config object to be injected in the html template.

Also expose two public directory:

  • the public directory which is exposed behind the root path
  • the ./.build/public directory which is exposed behind the build path

(async) start()

Source:
  • server/Server.js, line 440

Method to be called when init step is done in the initialization lifecycle of the soundworks server.

What it does:

  • starts all registered contexts (context are automatically registered when instantiated)
  • start the web socket server
  • launch the HTTP server on given port

After await server.start() the server is ready to accept incoming connexions
